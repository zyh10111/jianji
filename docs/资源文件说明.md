# 应用资源文件说明

本文档说明如何为简记（NoteEase）应用添加自定义图标和启动屏幕。

---

## 当前状态

当前项目**暂时移除了自定义图标配置**，使用 Expo 的默认图标。这样可以确保构建能够成功。

如果您想添加自定义图标，请按照以下步骤操作。

---

## 必需的资源文件

### Android 资源

1. **icon.png** - 主应用图标
   - 尺寸：1024x1024 像素
   - 格式：PNG
   - 位置：`assets/icon.png`

2. **adaptive-icon.png** - Android 自适应图标
   - 尺寸：1024x1024 像素
   - 格式：PNG
   - 位置：`assets/adaptive-icon.png`
   - 注意：图标主体应该居中，四周留出安全区域（约 20%）

3. **splash.png** - 启动屏幕图片
   - 尺寸：1242x2436 像素（推荐）
   - 格式：PNG
   - 位置：`assets/splash.png`

### iOS 资源

1. **icon.png** - 应用图标（同上）
2. **splash.png** - 启动屏幕（同上）

### Web 资源

1. **favicon.png** - 网站图标
   - 尺寸：256x256 像素（或更大）
   - 格式：PNG
   - 位置：`assets/favicon.png`

## 生成占位符图片（快速方案）

### 方法一：使用脚本自动生成

1. **安装依赖**（如果还没有安装 sharp）：
   ```bash
   npm install sharp --save-dev
   ```

2. **运行生成脚本**：
   ```bash
   node scripts/generate-assets.js
   ```

3. **恢复 app.json 配置**：
   取消注释 `app.json` 中的图标配置：
   ```json
   {
     "expo": {
       "icon": "./assets/icon.png",
       "splash": {
         "image": "./assets/splash.png",
         ...
       },
       "android": {
         "adaptiveIcon": {
           "foregroundImage": "./assets/adaptive-icon.png",
           ...
         }
       },
       "web": {
         "favicon": "./assets/favicon.png"
       }
     }
   }
   ```

### 方法二：使用在线工具生成

1. **生成图标**：
   - 访问 https://www.favicon-generator.org/
   - 或使用 Figma、Canva 等设计工具
   - 设计尺寸：1024x1024 像素

2. **生成启动屏幕**：
   - 使用设计工具创建 1242x2436 像素的图片
   - 确保重要内容在中心区域

3. **放置文件**：
   将生成的图片文件放到 `assets/` 目录

---

## 设计建议

### 图标设计

1. **简洁明了**
   - 使用简单的几何形状
   - 避免过多细节
   - 确保在小尺寸下也能清晰识别

2. **品牌一致性**
   - 使用统一的颜色方案
   - 保持与应用的视觉风格一致

3. **安全区域**
   - Android 自适应图标需要留出 20% 的安全区域
   - 重要内容不要放在边缘

### 启动屏幕设计

1. **简洁**
   - 避免过多文字
   - 使用品牌色和 Logo

2. **快速加载**
   - 使用简单的设计
   - 避免复杂的图片

## 恢复图标配置

在生成或添加图标文件后，需要恢复 `app.json` 中的配置：

```json
{
  "expo": {
    "icon": "./assets/icon.png",
    "splash": {
      "image": "./assets/splash.png",
      "resizeMode": "contain",
      "backgroundColor": "#ffffff"
    },
    "android": {
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#ffffff"
      }
    },
    "web": {
      "favicon": "./assets/favicon.png"
    }
  }
}
```

## 验证配置

添加图标后，验证配置是否正确：

```bash
# 检查 Expo 配置
npx expo config

# 应该不会显示图标文件找不到的错误
```

## 常见问题

### 1. 构建失败：找不到图标文件

**解决方案**：
- 确保 `assets/` 目录存在
- 确保文件名和路径完全匹配
- 检查文件扩展名（应该是 `.png`）

### 2. 图标显示不正确

**解决方案**：
- 检查图片尺寸是否正确
- 确保图片格式为 PNG
- 对于 Android 自适应图标，确保重要内容在中心 80% 区域内

### 3. 图标太模糊

**解决方案**：
- 使用高分辨率图片（至少 1024x1024）
- 使用矢量图或高分辨率位图
- 避免过度缩放

```
└── favicon.png           (256x256)   - Web 图标（可选）
```
